# asc-atac-analysis

Analysis pipeline for deeper analysis of scATAC-seq accessibility pseudobulk profiles across ASC metatypes.

## What This Repo Contains

- `analysis/`: core analysis scripts (project setup, peak calling, pseudobulking, marker modeling, chromVAR analyses)
- `figures/`: visualization and motif footprint plotting scripts/notebooks
- `helper_functions.r`: locally sourced helper functions (originally from ChAI)

## Prerequisites

R (>= 4.2 recommended) and the following packages:

- CRAN/Bioconductor: `ArchR`, `SummarizedExperiment`, `ComplexHeatmap`, `dplyr`, `tidyr`, `emmeans`, `GSVAdata`, `org.Hs.eg.db`, `TxDb.Hsapiens.UCSC.hg38.refGene`, `TxDb.Hsapiens.UCSC.hg38.knownGene`, `BSgenome.Hsapiens.UCSC.hg38`, `GenomicRanges`, `GenomicFeatures`
- GitHub/local: `MOCHA`

Notes:

- Scripts now source local helpers from `helper_functions.r` instead of importing ChAI directly.
- Some scripts use high core counts (`numCores` 20-40). Reduce if running locally.

## Required Input Data

Expected files/directories before running:

- `input/` containing `.arrow` files for ArchR
- `teaseq_4_05.rds` Seurat object with metadata columns used in pseudobulking and annotation

Intermediate outputs are written under run directories such as:

- `Plasma_ArchR/`
- `Run_Metatype_threshold/`

## Recommended Execution Order

1. `analysis/Step0_ArchR.R`
Purpose: initialize ArchR project and attach Seurat-derived cell metadata.

2. `analysis/Step1_scRNA_pseudobulking.R`
Purpose: create pseudobulk scRNA objects (isotype + meta) and genome-annotated versions.

3. `analysis/Step1_Calling_Peaks_in_scATAC.R`
Purpose: call open tiles, build SampleTileMatrix.

4. `analysis/Step2_MetaCluster_Markers.R`
Purpose: model scATAC markers and generate accessibility heatmaps.

5. `analysis/Step4_ChromVAR.r`
Purpose: perform chromVAR-based motif testing and cross-method motif comparisons.

6. `analysis/annotate_regions.r`
Purpose: annotate differential ATAC tiles with promoter/gene/exon/intron context.

## Helper Functions (No ChAI Import)

Scripts load helpers with a fallback pattern:

```r
helper_candidates <- c("helper_functions.r", "../helper_functions.r")
helper_path <- helper_candidates[file.exists(helper_candidates)][1]
if (is.na(helper_path)) {
	stop("Could not find helper_functions.r. Run from project root or script directory.")
}
source(helper_path)
```

If you add new scripts, use the same pattern to avoid hard dependency on ChAI package installation.

## Troubleshooting

- `Could not find helper_functions.r`: run script from repo root or adjust relative helper path.
- Missing object/file under run directories: execute scripts in the workflow order above.
- Out-of-memory during modeling/footprints: lower `numCores`, process smaller subsets, or skip optional footprint-heavy sections.
- Linux-only `prlimit` calls are wrapped with `try(...)` in cleaned scripts; safe to run on Windows.

## Reproducibility Tips

- Keep all paths relative to repo root.
- Save `sessionInfo()` for each run.
- Version-lock MOCHA/helper code when generating final figures/tables.
